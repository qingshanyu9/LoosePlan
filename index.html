核心功能

  • 人格化日程规划

    基于MBTI/性格/习惯的用户画像，生成专属日程方案，附带证据来源与权重解释，支持一键重排与影响分析。

  • 桌面+飞书双端联动

    Windows桌面挂件为主入口，飞书私聊为手机端镜像，消息会话流统一，离线状态仅桌面端可用本地功能。

  • 可编辑记忆图谱

    个人Wiki式记忆管理，AI自动提取习惯偏好，用户可自由增删改，与用户画像实时同步。

  • 日程可视化时间块

    按小时排布的周/日视图，支持任务/事件/时间块管理，包含重复规则、依赖关系、优先级等完整能力。

  • 自动订阅与每周回顾

    定时联网检索指定关键词内容，自动生成每周回顾报告，桌面端可查历史，飞书端接收摘要。

  • 纯本地数据管理

    所有数据（聊天/日程/记忆）本地JSON/Markdown存储，支持一键导出导入，完全掌控数据安全。

产品特色

  • 证据化建议体系

    所有日程建议均附带归一化到100%的证据权重，明确来源（用户画像/记忆/聊天/历史）与取舍逻辑。

  • 全流程变更审计

    所有日程/记忆/画像变更均记录JSON Patch与人类可读摘要，支持追溯与回放。

  • 高度自定义体验

    桌面挂件位置/大小/透明度可调整，全局快捷键自定义，数据目录可迁移，完全适配个人使用习惯。

  • 极简初始化引导

    首次启动完成画像问答即可生成专属报告，未完成初始化时飞书端拒绝服务，保证核心体验。

  • 轻量化部署

    无数据库依赖，仅需Python+PyQt5运行环境，安装包支持开机自启、卸载，适配Windows 10全版本。

  • 灵活的导出导入

    导出zip可选择是否包含敏感密钥，导入自动解压适配，轻松迁移至新电脑，数据不丢失。

LoosePlan 使用文档

一、安装与初始化

1. 下载GitHub Release中的Inno Setup安装包，运行后选择安装目录（建议默认），勾选“开机自启”选项。

2. 首次启动需完成初始化流程：

  • 确认数据目录（默认%LOCALAPPDATA%\LoosePlan\），支持后续在设置页修改（自动复制旧数据）。
  • 配置Kimi API Key（必填），飞书App ID/Secret为可选（仅启用手机端需要）。
  • 完成机器人称呼与用户称呼配置，参与15~20题用户画像问答（非直接询问MBTI）。
  • 生成用户画像报告并写入本地，未完成初始化将无法使用后续功能。

二、核心功能使用

1. 桌面挂件操作：

  • 默认显示在桌面右下角，支持拖拽、贴边吸附，全局快捷键Alt+Space快速打开聊天页。
  • 点击挂件展开功能环：聊天/日程/记忆图谱/每周回顾/设置/退出/隐藏。
  • 聊天页关闭时收到新消息会显示角标提醒，打开聊天页自动清零。

2. 日程管理：

  • 日程页左侧显示月历、计划统计、待办/成果，右侧为时间块视图（周/日切换）。
  • 支持添加任务/事件/时间块，设置重复规则、依赖关系、优先级、截止时间。
  • 点击“一键重排”可自动调整日程，查看延期风险、冲突时间块与替代方案。

3. 记忆图谱：

  • 左侧为分类导航（用户画像/日程偏好/行为习惯等），右侧为条目编辑区。
  • AI自动提取的记忆条目支持手动编辑，可添加来源、关联场景、修改时间。
  • 支持导入/导出记忆数据，所有修改均记录变更日志。

4. 飞书联动：

  • 仅支持1v1私聊，首次收到飞书消息时绑定Owner OpenID，仅处理该用户消息。
  • 桌面端离线时飞书端不可用，消息与桌面端合并为同一条会话流。

三、数据管理

1. 导出/导入：

  • 在设置页点击“一键导出”，选择是否包含Kimi/飞书密钥，生成zip包存储在export目录。
  • 导入时选择zip包，自动解压到当前数据目录，重启后生效。

2. 变更日志：

  • 所有日程/记忆/订阅/画像变更均写入schedule/changes.jsonl，包含时间戳、操作人、摘要、JSON Patch。
  • 日志支持审计与回放，可追溯任意数据修改记录。

四、自动订阅与每周回顾

1. 自动订阅：在设置页配置关键词、检索频率、发送时间，结果自动写入订阅库并去重，支持桌面/飞书推送。

2. 每周回顾：按设置时间自动生成Markdown（详细）+JSON（概括）文件，桌面端可查历史，飞书端仅接收摘要，离线时跳过当次生成。

五、常见问题

Q1: 飞书端无法接收消息？

A1: 检查桌面端是否运行、飞书App ID/Secret配置是否正确、是否已绑定Owner OpenID，未完成初始化也会导致飞书端拒绝服务。

Q2: Kimi调用失败怎么办？

A2: 检查API Key是否有效，网络是否正常，失败时会提示重试，且不会写入日程变更。

Q3: 如何迁移数据到新电脑？

A3: 在旧电脑导出zip包（建议不包含密钥），新电脑安装后完成初始化，再导入zip包即可。
